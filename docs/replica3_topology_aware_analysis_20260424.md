# 三副本拓扑感知放置分析

## 分析范围

本文只讨论当前 7 个拓扑在 `3-Replica Topology-Aware` 场景下的选点思路和结果差异：

- `Clos`
- `2D-FullMesh`
- `2D-Torus`
- `2D-Torus-BestTwist`
- `3D-Torus`
- `3D-Torus-BestTwist`
- `DF`

## 核心结论

1. 当前三副本方案不是严格证明的全局最优，而是一个按拓扑结构设计的对称近优方案。

2. `2D-FullMesh`、`3D-Torus`、`3D-Torus-BestTwist` 是当前三副本场景的第一档，吞吐均为 `400 Gbps`。

3. `2D-Torus`、`2D-Torus-BestTwist`、`DF` 为第二档，吞吐均为 `266.67 Gbps`。

4. `Clos` 在三副本场景下最低，为 `200 Gbps`，说明它更适合 `A2A` 这类全局均匀流量，而不是“每源两个定向副本”的流量模式。

5. `BestTwist` 对 `A2A` 提升明显，但对当前三副本场景几乎没有吞吐提升，说明三副本的瓶颈主要在目标选择模式，而不是 Torus 的长程布线方式。

## 各拓扑的三副本选点方式

### 2D-FullMesh

- 一个副本放到同一行的其他 exchange
- 一个副本放到同一列的其他 exchange
- 不同源通过轮转方式错开具体目标

特点是两个副本天然走两类不同方向，既短又容易分流，因此非常适合三副本场景。

### 2D-Torus / 2D-Torus Best Twist

- 两个副本优先从四个一跳邻居 `+X/-X/+Y/-Y` 中选择
- 不同源按编号轮转方向，尽量把压力摊开

它的优点是副本目标很近，但二维方向数有限，两个副本仍容易在少数方向上形成竞争。

### 3D-Torus / 3D-Torus Best Twist

- 两个副本优先从六个一跳邻居 `±X/±Y/±Z` 中选择
- 同样通过轮转方式做全网分散

相比 2D-Torus，3D-Torus 的主要优势是可选方向更多，因此两个副本更容易被摊到不同维度，热点更轻。

### Clos

- Clos 没有天然“近邻”概念
- 因此两个副本不是按几何最近去挑，而是按全局分散原则去挑目标 exchange

它更像“均衡分流”，而不是“就近放置”。这对 A2A 很有效，但对三副本这种强局部性需求的业务不占优。

### Dragon-Fly

- 一个副本放在同一本地 group 内
- 另一个副本放在可直接到达的远端 group 上
- 不同源再轮转使用具体方向

这种方式很好地利用了 DF 的 local/global 分层结构，因此比随机选点合理得多，但灵活性仍不如 `2D-FullMesh` 和 `3D-Torus`。

## 是否为全局最优

严格来说，不构成。

原因是当前方法并不是把“全网所有源的两个副本目标”放在同一个联合优化问题里求解，而是：

- 先为每个源按拓扑规则选两个较优目标
- 再依靠拓扑对称性和轮转机制，让全网尽量均衡

因此它的定位应当是：

- 对单个源，通常是最短或近最短路径
- 对全网，是结构感知、可解释、可扩展的近优方案
- 但不是经过全局穷举或数学证明的最优解

## 为什么与A2A仍有差距

三副本最优方案与 `A2A` 有差距是正常现象，因为两者对应的是两类完全不同的流量模式。

`A2A` 的特点是：

- 每个源对全网所有目标发流
- 更考验对分能力和长程均衡

三副本的特点是：

- 每个源只打两个目标
- 更依赖局部选点是否合理
- 更依赖这两个目标是否会在全网形成重复热点

因此：

- 如果某拓扑能把两个副本都放到短路径上，三副本吞吐就可能高于 `A2A`
- 如果两个副本虽然近，但全网都偏向同几类方向，三副本吞吐仍会被热点限制

换句话说，三副本的关键不是“流量少”，而是“这两个目标是否选得又近又散”。

## 拓扑间结果对比

### 第一档：2D-FullMesh与3D-Torus

这三类拓扑的共同点是：

- 两个副本都能压到较短路径
- 两个副本之间又能较好地分散到不同方向

其中：

- `2D-FullMesh` 依赖行列两个 clique 天然分流
- `3D-Torus` 依赖三维方向自由度分流

所以它们都能达到 `400 Gbps`。

### 第二档：2D-Torus与DF

这三类拓扑都能利用结构做“较近的副本放置”，但仍受限于方向或层次结构本身：

- `2D-Torus` 的问题是二维方向太少
- `DF` 的问题是 local/global 结构固定，目标选择自由度不够高

因此它们都停留在 `266.67 Gbps`。

### 第三档：Clos

`Clos` 在这组对比里最弱，不是因为它不均衡，而是因为它缺乏“近邻副本”优势。

对于三副本这种业务，理想目标是：

- 副本尽量近
- 两个副本又尽量分散

Clos 更擅长的是“全网任意对都能稳定承载”，这更接近 `A2A` 的需求，而不是当前这种副本业务。

## Best Twist的作用边界

当前结果很清楚：

- `2D-Torus -> 2D-Torus-BestTwist`，`A2A` 提升明显，但三副本吞吐不变
- `3D-Torus -> 3D-Torus-BestTwist`，`A2A` 提升明显，但三副本吞吐不变

这说明 `BestTwist` 主要解决的是：

- 全网均匀大流量下的长程热点问题

而当前三副本场景的主导因素是：

- 两个副本目标是否选得合理

所以对三副本而言，继续优化目标选择策略，通常比继续优化 Torus 的 twist 方式更有价值。

## 最终结论

- 当前三副本方案可以视为“拓扑感知的近优副本放置”。
- `2D-FullMesh` 和 `3D-Torus` 最适合当前三副本业务。
- `2D-Torus` 和 `DF` 次之，说明它们能利用局部结构，但自由度仍不足。
- `Clos` 更适合 `A2A`，不适合当前这种定向三副本流量。
- `BestTwist` 的主要价值体现在 `A2A`，而不是当前三副本场景。
